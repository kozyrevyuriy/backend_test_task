# Backend Authentication & Authorization System

Тестовое задание на позицию Junior Backend Python Developer.

## Описание

Разработано backend-приложение с собственной системой аутентификации и авторизации пользователей.

Реализованы:

* регистрация пользователя;
* вход в систему (login);
* выход из системы (logout);
* просмотр и редактирование профиля;
* мягкое удаление пользователя (soft delete);
* система ролей и прав доступа;
* разграничение доступа к ресурсам;
* обработка ошибок 401 и 403.

---

## Используемые технологии

* Python
* Django
* Django REST Framework
* PostgreSQL (или SQLite для локального запуска)

---

## Схема БД

### User

Хранит данные пользователя.

Поля:

* id
* email
* password_hash
* first_name
* last_name
* middle_name
* is_active

---

### Role

Хранит роли пользователей.

Примеры:

* admin
* user

Поля:

* id
* name

---

### Permission

Хранит разрешения.

Примеры:

* view_resource
* edit_resource
* manage_permissions

Поля:

* id
* name

---

### UserRole

Связь пользователей и ролей.

Поля:

* user_id
* role_id

---

### RolePermission

Связь ролей и разрешений.

Поля:

* role_id
* permission_id

---

### UserSession

Хранение пользовательских сессий.

Поля:

* id
* user_id
* refresh_token_hash
* created_at
* expires_at
* is_revoked

---

## Логика аутентификации

1. Пользователь проходит регистрацию.
2. Выполняет вход по email и паролю.
3. Система создаёт пользовательскую сессию.
4. При последующих запросах пользователь идентифицируется по сессии.
5. После logout сессия завершается.

---

## Логика авторизации

1. Система определяет пользователя.
2. Получает роли пользователя.
3. Получает разрешения, назначенные этим ролям.
4. Проверяет наличие требуемого разрешения.
5. Если разрешение есть — доступ предоставляется.
6. Если пользователь не определён — возвращается HTTP 401.
7. Если пользователь определён, но прав недостаточно — возвращается HTTP 403.

---

## Управление доступом

Администратор может:

* просматривать роли;
* изменять роли пользователей;
* назначать разрешения ролям;
* изменять правила доступа.

## Тестирование

Проверены следующие сценарии:

* успешная регистрация;
* успешный вход;
* доступ авторизованного пользователя к защищённым ресурсам;
* получение ошибки 401 для неавторизованного пользователя;
* получение ошибки 403 при отсутствии необходимых прав;
* мягкое удаление пользователя;
* запрет входа после деактивации аккаунта.
